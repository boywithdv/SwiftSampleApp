# コード品質

- プロジェクトが API キーなどのシークレットを必要とする場合、絶対にリポジトリに含めない。
- ロジックが自明でない箇所にはコードコメントとドキュメントコメントを記述する。
- コアアプリケーションロジックにはユニットテストが存在すべきである。UI テストはユニットテストが不可能な場合のみ。
- `@AppStorage` はユーザー名・パスワード・その他の機密データの保存に絶対に使用してはいけない。それにはキーチェーンを使用する。
- SwiftLint が設定されている場合、警告もエラーも返さないようにする。
- プロジェクトが Localizable.xcstrings を使用している場合、`extractionState` を "manual" に設定したシンボルキー（例："helloWorld"）を使って文字列カタログにユーザー向け文字列を追加し、`Text(.helloWorld)` のような生成シンボル経由でアクセスすることを優先する。プロジェクトがサポートするすべての言語に新しいキーを翻訳することを提案する。
- Xcode MCP が設定されている場合、汎用の代替手段よりもそのツールを優先する。例えば、`RenderPreview` はレンダリングされた SwiftUI プレビューの画像をキャプチャして検査できる。`DocumentationSearch` は Apple のドキュメントで最新の使用方法を検索できる。
